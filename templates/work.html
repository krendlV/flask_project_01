Aufgrund von Komplikationen mit der implementierung der Register-User Funktion haben wir uns dazu entschieden die USer mit SQLAlchemy anzulegen da wir folgende Tutorials dazu gefunden haben: LINK User Authentication & LINK Database in Flask SQLAlchemy hat eine sehr hohe Lernkurve. Leider hat sich ein Error durchgezogen, welcher das schreiben in der Datenbank inklusive Login, nur unter gewissen umständen Erlaubt hat.

Die Datenbankimplementierung wurde anfangs mit einem Schema.sql und einem sql File zum befüllen der Datenbank über das init_db.py File gemacht. Diese Dateien sind der Dokumentation halber noch vorhanden.
Die Schlussendliche implementierung erfolgt über SQLAlchemy-Klassen in einer models.py Datei, welche über eine Funktion zum erstellen der Datenbank aufgerufen wird. Wenn die app.py aufgerufen wird wird die Datenbank erstellt und mit Daten befüllt. Wenn sie bereits existiert wird sie nicht neu erstellt und Flask ist startbar. Die Funktion zum befüllen mit Testdaten befindet sich in testdata.py. Die Funktion könnte sehr minimiert geschrieben werden. der Vorteil dieser Methode (commit nach jedem table) ist, dass der Code leicht änderbar und wiederverwendbar ist.
